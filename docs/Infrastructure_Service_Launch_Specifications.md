
# 基础设施服务上线流程规范  

## 1、 目的概述  
本规范旨在为开源社区基础设施服务上线发布提供标准化、流程化指导，向服务开发者提供明确流程步骤说明，对服务运维者提供标准的操作规范。  

## 2、 规则适用范围  
本规范适用于openEuler社区基础设施服务（包括软件开发类、用户服务类、社区运营类）上线流程管理。

## 4、 流程节点及说明  

| 流程         | 序号 | 流程细节说明                                                                 | 责任方               |
|--------------|------|------------------------------------------------------------------------------|----------------------|
| 需求评审     | 1    | 内容准备：开发输出服务上线checklist                                          | 开发团队             |
| 需求评审     | 2    | 需求评审：对checklist进行评审，包含checklist疑问点、服务部署框架，补充输出评审纪要 | 运维团队、开发团队   |
| 安全评审     | 1    | 安全评审：根据业务暴露风险完成对服务上线的分级安全评审，输出评审纪要。       | 开发团队             |
| 服务预发布流程 | 1    | 服务部署预上线                                                               | 运维团队             |
| 服务预发布流程 | 2    | 验收标准：服务上线效果与需求评审通过的方案安全一致                           | 运维团队、开发团队   |
| 服务正式发布 | 1    | 部署文件归档，输出CI/CD流水线，根据开发团队需求确认是否需输出适当容器访问权限，服务接入监控管理 | 运维团队             |

## 5、 责任主体及评审接口人  

| 流程节点     | 责任主体       | 职责要求                                                                 | 评审接口人                           |
|--------------|----------------|--------------------------------------------------------------------------|--------------------------------------|
| 需求评审     | 开发团队       | 评审服务是否满足基础设施上线规则（基础镜像国产化、无明显安全漏洞、checklist信息完善），评估服务部署架构 | 曹志[georgecao](https://atomgit.com/georgecao)、钟君 [zhongjun2](https://atomgit.com/zhongjun2)、刘洋[liuyanglinux](https://atomgit.com/liuyanglinux) |
| 安全评审     | 开发团队       | 评审服务上线可能的安全风险，是否完成服务注册，服务风险排查，服务送检等一些列动作，并获得“三证一码”。 | 各业务部门安全评审组织，产品线IT装备安全SUB_TMG, 基础设施安全SE 赵昆航[zkhzkhz](https://atomgit.com/zkhzkhz) |
| 服务预发布   | 运维团队、开发团队 | 运维团队部署服务预上线，开发团队验收上线是否满足需求评审会议要求         | 刘洋[liuyanglinux](https://atomgit.com/liuyanglinux)                       |
| 服务正式发布 | 运维团队       | 部署文件归档，CI/CD流水线创建，根据需求评审纪要，输出开发需要的容器访问权限，接入运维监控平台 | 刘洋[liuyanglinux](https://atomgit.com/liuyanglinux)                       |

## 附件  

### 服务上线checklist（摘要）  
| 检查项               | 确认结果 | 备注                                                                 |
|----------------------|----------|----------------------------------------------------------------------|
| 服务类别             |          | 前端/后端                                                            |
| 服务介绍             |          | 简单描述该服务的功能                                                 |
| Dockerfile           |          | Gitee/github/AtomGit仓库地址                                                 |
| 容器基础镜像         |          | 是否以openEuler作为基础镜像（如未使用请说明修改计划）                |
| CPU架构              |          | arm/x86                                                              |
| 端口                 |          | 服务对外暴露端口                                                     |
| 环境变量/配置文件    |          | 安全需要，建议使用环境变量或配置文件挂载                             |
| 域名                 |          | 安全需要，使用域名而非IP+端口                                        |
| 容器状态             |          | 需提供自验证报告                                                     |
| 健康检测             |          | 提供快速检测方法                                                     |
| 容器基础资源         |          | CPU、内存需求                                                        |
| 区域                 |          | 社区+区域（如：openEuler 北京 生产环境）                             |
| 持久存储             |          | 如需持久存储，说明路径和空间大小（如：/var/log 10G）                 |
| 数据库               |          | 数据库配置（如：MySQL 8.0 字符集UTF8）                               |
| 通信                 |          | 关联服务的地域、IP或域名                                             |
| 安全自检             |          | 防暴力破解、端口暴露、网络隔离等                                     |
| 调试权限申请         |          | 日志查看、容器exec、端口映射等                                       |
| API监控              |          | 需监控的API URL和参数                                                |
| 串讲纪要             |          | 会议纪要、参与人、日期、遗留问题                                     |
| 上线时间             |          | 服务上线时间节点                                                     |
| 责任人               |          | 服务联系人                                                           |
